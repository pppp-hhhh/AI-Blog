# 首页文章与其他页面不一致问题分析报告

## 问题发现

通过代码分析和数据对比，发现了首页文章与其他页面不一致的根本原因：

## 主要问题

### 1. 分类数据不一致

**首页** (`index.vue`)：使用硬编码的分类数据，显示的是静态的、不准确的文章数量：
```javascript
const categories = [
  { name: '前端开发', slug: 'frontend', count: 12 },
  { name: '后端开发', slug: 'backend', count: 8 },
  { name: '技术随笔', slug: 'thoughts', count: 15 },
  { name: '工具推荐', slug: 'tools', count: 6 }
]
```

**实际数据**（根据文章文件统计）：
- frontend: 2篇
- css: 1篇  
- performance: 1篇

**博客页面** (`blog/index.vue`)：使用动态数据，根据实际文章进行分类统计

### 2. 文章获取逻辑分析

**首页**：获取最新6篇文章
```javascript
const { data: recentPosts } = await queryContent('/articles')
  .sort({ date: -1 })
  .limit(6)
  .find()
```

**博客页面**：获取所有文章，然后在前端进行筛选和分页
```javascript
const { data: posts } = await queryContent('/articles')
  .sort({ date: -1 })
  .find()
```

### 3. 数据一致性验证

根据实际文章文件分析：

**文章列表**：
1. 现代CSS布局技术完全指南 (2024-02-01) [css]
2. 前端性能优化实战指南 (2024-01-25) [performance]
3. Vue 3组合式API完全指南 (2024-01-20) [frontend]
4. 文章标题 (2024-01-15) [frontend]

**首页显示**：正确显示最新6篇文章（实际只有4篇）
**博客页面显示**：正确显示所有4篇文章

## 问题影响

1. **分类统计不准确**：首页显示的分类文章数量与实际严重不符
2. **用户体验问题**：用户可能认为博客内容不完整或存在bug
3. **数据可信度下降**：统计数据不准确影响博客的专业性

## 解决方案

### 方案1：修复首页分类数据（推荐）
将首页的分类数据改为动态计算，与实际文章数据保持一致

### 方案2：统一数据获取逻辑
确保所有页面使用相同的数据获取和统计方法

## 建议优先级

1. **高优先级**：修复首页分类统计数据的准确性
2. **中优先级**：考虑统一所有页面的数据展示逻辑
3. **低优先级**：优化文章获取性能
